# 題庫清單改版 — 前端對接說明

> 對象：前端。後端已上線下列變更，請依「前端要做的事」調整。

## 一、這次改了什麼（後端）

1. **清單放寬**：`GET /v1/bank/pages?course_id=<id>` 現在回傳該課程**所有人**建立的題庫（不再只限呼叫者本人）。前端**維持原樣**：只帶 `course_id` + token，無新參數。
2. **新增建立者/修改者欄位**：每筆 `banks[]` 多了 `creator` / `creator_name` / `updater` / `updater_name`。
3. **權限改為課程共管**：同課程**管理者（user_type 1 管理者 / 2 教師）**可設定/編輯/上傳/出題/批改/刪除**任何**題庫；不再「限本人」。學生（type 3）只能讀。

---

## 二、新欄位（`GET /v1/bank/pages` 回應）

每筆 `banks[]` 物件新增：

| 欄位 | 型別 | 說明 |
|---|---|---|
| `creator` | string | 建立者 `person_id`，**永不變** |
| `creator_name` | string | 建立者顯示名（找不到時為 `""`） |
| `updater` | string | 最後修改任何欄位（含出題/編輯 prompt/批改/刪題組）的人的 `person_id` |
| `updater_name` | string | 最後修改者顯示名（找不到時為 `""`） |
| `person_id` | string | **保留**，等於 `creator`（前端舊邏輯可繼續用） |

範例：

```json
{
  "banks": [
    {
      "bank_id": 1,
      "bank_page_id": "abc",
      "person_id": "u_123",
      "creator": "u_123",
      "creator_name": "王小明",
      "updater": "u_456",
      "updater_name": "李大華",
      "course_id": 10,
      "tab_name": "第一章",
      "units": [ /* ... 原樣 ... */ ]
    }
  ],
  "count": 1
}
```

> 名稱可能為空字串（成員資料缺失時），UI 請做好 fallback（例如空就只顯示單元數）。

---

## 三、前端要做的事

### ✅ 1. 顯示建立者（要改）

- 清單每列副標題接上 `creator_name`，例如：`3 個單元 · 建立者：王小明`。
- 可選：一併顯示最後修改者，例如 `建立者：王小明 · 最後修改：李大華`。
- 「我的」標示：用 `bank.creator === 目前登入者 person_id`（等同 `bank.person_id`）。
- 影響檔案（依你方盤點）：`src/composables/useQuizBankListPage.js` 的 `gridItems` / `ragSubtitle`、`src/pages/BankPage.vue`。

### ✅ 2. 權限按鈕（要改，且和原規劃不同）

原本規劃是「編輯/刪除限本人，依 `person_id` 隱藏按鈕」——**作廢**。最終是**管理者共管**：

- **不要**再用「`creator === 我`」來隱藏編輯/上傳/出題/刪除按鈕。
- 改用「**我是不是課程管理者（user_type 1/2）**」來決定是否顯示寫入按鈕。
  - 若前端拿得到登入者在該課程的 `user_type`：`user_type ∈ {1,2}` 才顯示寫入操作。
  - 若前端拿不到 `user_type`：可**不隱藏**，直接送請求，由後端擋（非管理者回 **403**），再用錯誤訊息提示。
- 學生（type 3）觸發寫入 → 後端回 `403`，請顯示「僅課程管理者可操作」之類訊息。

### ❌ 3. 清單放寬本身（不用改）

後端給什麼就顯示什麼，query 參數不變。

---

## 四、權限對照（後端行為）

| 操作 | 誰可以 |
|---|---|
| 列清單 / 看詳情 / 看單元內容 | 課程內任何有效登入者（清單回全課程題庫） |
| 建立 / 改名 / 上傳 / 建置(build-zip) / 預覽 | 課程管理者（1/2），可對任何題庫 |
| 出題 / 重出 / 編輯 prompt / 設定 for_exam / 批改 | 課程管理者（1/2），可對任何題庫 |
| 刪題庫 / 刪題組 / 刪題目 | 課程管理者（1/2），可對任何題庫 |

非管理者執行寫入 → `403`；找不到/已刪 → `404`。

---

## 五、注意事項

- `updater` 會隨任何內容變更而更新（含出題、批改、編輯、刪子項），清單重新整理後 `updater_name` 會反映最新修改者；`creator` 永遠不變。
- 詳情頁（`GET /v1/bank/pages/{bank_page_id}/units` 等）本來就能讀別人的題庫，無需改動。
